# main-wd-pages

本仓库用于归档与发布多个独立 HTML 页面，作为页面资源的集中存储与静态展示场所。不针对任何特定域名或网站，仅提供页面内容的版本管理与访问支持。

## 项目简介

本项目是一个通用型 HTML 页面归档仓库，适合存放各类独立的、无需后端支持的静态页面。页面内容以纯前端形式存在，可通过 GitHub Pages 或其他静态托管服务直接访问。

## 目录说明

```
main-wd-pages/
├── pages/          # 存放各个独立 HTML 页面
├── assets/         # 公共资源文件（如 CSS、JS、图片等）
├── index.html      # 仓库首页（可选）
└── README.md       # 本文件
```

- **pages/**：每个子目录或独立 HTML 文件代表一个归档页面。
- **assets/**：页面间共享的样式、脚本或媒体文件。
- **index.html**：仓库根目录下的入口页面，可根据需要创建或省略。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，不依赖外部数据库或后端服务。
- 页面内容可包括文档、工具、演示、实验性项目等，不设主题限制。
- 归档时建议在页面内注明创建日期、版本或简要说明，便于后续维护。
- 如需引用外部资源（如字体、库文件），建议使用可靠的 CDN 或本地存放。

## 维护说明

- 欢迎通过 Issue 或 Pull Request 提交页面更新、修复或新增归档。
- 提交前请确保页面内容合法、合规，且不包含恶意代码或敏感信息。
- 建议保持目录结构清晰，避免文件命名冲突。
- 本仓库不提供自动部署配置，如需启用 GitHub Pages，请自行在仓库设置中开启。
